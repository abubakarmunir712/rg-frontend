# GitHub Push Guide

## Quick Reference for Pushing to GitHub

### First Time Setup

1. **Create a new repository on GitHub** (don't initialize with README, .gitignore, or license)

2. **Initialize git locally** (if not already done):
```bash
git init
```

3. **Add all files**:
```bash
git add .
```

4. **Create initial commit**:
```bash
git commit -m "Initial commit: Research Gennie Frontend"
```

5. **Add remote repository**:
```bash
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
```

6. **Push to GitHub**:
```bash
git branch -M main
git push -u origin main
```

### Subsequent Pushes

1. **Check status**:
```bash
git status
```

2. **Add changes**:
```bash
git add .
# or add specific files
git add src/components/NewComponent.tsx
```

3. **Commit changes**:
```bash
git commit -m "feat: add new feature"
```

4. **Push to GitHub**:
```bash
git push
```

### Useful Git Commands

**View commit history**:
```bash
git log --oneline
```

**View changes**:
```bash
git diff
```

**Undo last commit (keep changes)**:
```bash
git reset --soft HEAD~1
```

**Create a new branch**:
```bash
git checkout -b feature/new-feature
```

**Switch branches**:
```bash
git checkout main
```

**Merge branch**:
```bash
git checkout main
git merge feature/new-feature
```

### What's Ignored

The following are automatically ignored by `.gitignore`:
- `node_modules/` - Dependencies
- `dist/` - Build output
- `.env` - Environment variables (sensitive data)
- `*.log` - Log files
- `.vscode/` - Editor settings (except extensions.json)
- Coverage reports
- OS-specific files (.DS_Store, Thumbs.db)

### What's Included

The following important files ARE included:
- `.env.example` - Template for environment variables
- All source code in `src/`
- Configuration files (vite.config.ts, tsconfig.json, etc.)
- Documentation (README.md, SETUP.md, CONTRIBUTING.md)
- LICENSE
- `.gitignore` and `.gitattributes`

### Before Pushing

Always verify:
1. ✅ No sensitive data in commits (API keys, passwords, etc.)
2. ✅ Tests pass: `npm run test:run`
3. ✅ Build succeeds: `npm run build`
4. ✅ No unnecessary files included
5. ✅ Commit messages are clear and descriptive

### Troubleshooting

**If you accidentally committed sensitive data**:
```bash
# Remove file from git but keep locally
git rm --cached .env

# Commit the removal
git commit -m "Remove .env from tracking"

# Push changes
git push
```

**If you need to change the last commit message**:
```bash
git commit --amend -m "New commit message"
git push --force
```

**If you want to see what will be pushed**:
```bash
git diff origin/main..HEAD
```

### GitHub Repository Settings

After pushing, consider:
1. Add repository description
2. Add topics/tags (react, typescript, vite, tailwindcss)
3. Enable GitHub Pages (if deploying)
4. Set up branch protection rules
5. Configure GitHub Actions for CI/CD
